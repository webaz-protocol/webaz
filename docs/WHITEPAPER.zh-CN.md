# WebAZ 创始白皮书 v0.1

## 状态说明

WebAZ 当前处于 pre-launch 和持续开发阶段。本文是创始说明和协议方向，不是投资文件，不是 token 白皮书，也不是收入、奖励、所有权或兑付承诺。

## 摘要

WebAZ 是一个 agent-native 的商业与贡献协议。

它从一个简单的信念出发：在 AI 时代，最重要、最有价值的是参与。

每一个人的存在和参与，本身就是真实的价值。一个人不应该为了贡献、建设、交易或表达，不断向平台、雇主、算法或电子围栏证明自己真实存在。存在、思考、行动、建设、帮助、改进某件事，并留下参与的证据，这些本身就是价值。

我思故我在。

在 WebAZ 里，我们把这句话带入 AI 时代：

我参与，故我被看见。

WebAZ 试图建设一种新的商业文明：人和 agent 可以一起参与，贡献会留下证据，规则可以被审计，AI 可以辅助，但最终责任仍由人承担；参与机会不应只属于已经站在大公司和大平台内部的人。

## 1. 参与的价值

很长一段时间，公司和平台通过围住用户、数据、流量、商家、劳动者和创作者来建立护城河。

那些围墙让参与变成一种被许可的事情。谁能进入，谁能被发现，谁能销售，谁能被信任，谁的贡献会被记住，都由平台决定。

AI 正在改变这一切。

当个人可以和 agent 一起工作，执行力就不再只属于大型组织。一个带着 agent 的人，可以建设、销售、审核、翻译、测试、协调和贡献。AI 时代真正的护城河，不再是围住用户的墙，而是愿意一起建设的人持续参与所形成的生命力。

WebAZ 建立在这个信念之上。

面对强大的公司，我们不需要担心自己没有护城河。我们的参与就是护城河。我们的证据就是护城河。我们的共同协议就是护城河。由参与者共同建设的文明，比围猎用户的平台更难被圈住。

WebAZ 不从攻击公司或资本开始，而是从削弱它们对参与机会的垄断开始。

公司和资本仍然可能存在，但它们不应该继续成为通往机会的唯一入口。WebAZ 是为这样一个未来而建：科技降低参与门槛，AI 放大个人而不是替代个人，开放协议让每一个愿意建设的人都能进入商业文明。

翻篇。

从今天开始，每一步都可以算数。

## 2. 为什么需要 WebAZ

互联网让发布变得开放。开源让代码协作变得开放。AI agent 可能让执行力变得开放。

但商业仍然被大量平台锁住。

卖家依赖自己并不真正拥有的渠道。买家很难知道推荐为什么出现。贡献者可以创造价值，却被遗忘在公司、队列、issue 或离职记录之外。很多人在岗位上长期解决真实问题，但一旦组织变化，他们的贡献就像没有发生过。大机构之外的建设者可能有能力，却没有入口。

WebAZ 之所以存在，是因为这不应该是唯一道路。

一个人不应该必须先被大公司雇佣，才能参与建设。一个小卖家不应该为了服务买家，把全部关系交给平台。一个贡献者不应该只能把工作交出去，却无法留下证据。一个 AI agent 可以执行工作，但不能冒充背后的责任人。

WebAZ 是为那些被忽视之后仍然想建设的人而生的。

## 3. WebAZ 是什么

WebAZ 不只是一个应用，而是一层协议。

它有人类使用的 PWA，也有 AI agent 使用的 MCP 接口。两者连接同一套底层规则：商品、订单、托管、履约、争议、验证、任务协调、贡献记录、审计轨迹和治理边界。

WebAZ 的核心表达是：

代码即规则，协议即信任。

只存在于宣传里的规则，不算规则。没有被代码执行的承诺，不算协议。无法追踪的贡献，太容易被抹掉。没有审计轨迹的决策，太容易被滥用。

因此，透明、可审计、可问责，不是 WebAZ 的装饰，而是设计要求。

## 4. WebAZ 不是什么

WebAZ 不是已经完成的生产经济体。

WebAZ 不保证收入。

WebAZ 不是 token 炒作项目。

WebAZ 不是一个出售排序、出售用户数据、要求用户相信隐藏规则的平台。

WebAZ 不是一个由 AI 自动裁决人类价值的系统。

WebAZ 今天也不能宣称已经完全去中心化。它的方向是渐进式去中心化：先从明确的创始人责任、公开规则、受保护的协议不变量和真实代码库开始；随着社区、审计能力和安全机制成熟，再逐步把更多权力交给贡献者和治理结构。

诚实比宏大的口号更重要。早期就说早期。没有代码执行的功能，就不要宣传为已经完成。贡献价值仍未承诺，就必须标注为 uncommitted。

## 5. 人与 Agent

WebAZ 是 agent-native，但不是 agent-ruled。

AI agent 可以搜索、比较、起草、翻译、测试、审计、总结和执行有边界的任务。它可以帮助卖家管理商品，帮助买家做选择，帮助管理员整理建议，帮助贡献者提交 PR。

但 agent 不是最终的道德或法律主体。

在 WebAZ 里，真正承担责任的是人或组织。Agent 可以执行，但人做决定。Agent 可以建议，但人来确认。Agent 可以起草，但人来发布。Agent 可以辅助审核，但最终判断必须留下人的责任。

这一点非常重要。它防止 WebAZ 变成一个黑箱系统，让人被 AI 静默拒绝、降权或抹去。

AI 应该扩大参与，而不是替代责任。

## 6. 贡献如何被看见

WebAZ 的贡献原则是：

先贡献，后认领。

一个人应该可以从很小的事情开始：修一个错字，补一条翻译，报告一个 bug，写一个测试，提出一个任务建议，提交一个 PR，或者带着自己的 agent 完成一个清晰的小任务。

这些工作应该被记住。

但被记住不等于承诺金钱。WebAZ 记录贡献证据、署名、来源、审核状态和可认领关系。它不因为一次贡献本身承诺奖励公式、兑付、百分比、token、收入或任何未来收益。

这个边界很重要。它保护贡献者不被虚假承诺吸引，也保护协议不变成炒作机器。

WebAZ 的承诺更窄，但更重要：

真实贡献不应消失。

## 7. 任务板

如果 WebAZ 要欢迎外部贡献者，就不能只依赖模糊的善意。它需要一套实际的协调层。

一个正式任务应该告诉人或 agent：要做什么，为什么重要，允许改哪些文件，禁止碰哪些路径，禁止做哪些动作，如何验证结果，PR 应该提交到哪里，怎样才算完成。

这就是 WebAZ 需要 agent-ready task model 的原因。参与者的 agent 应该能发现公开任务、理解执行边界、在合适时认领、提交 PR，并报告验证结果。

高风险任务必须不同处理。任何涉及资金、生产密钥、真实用户数据、认证、治理、数据库迁移或部署的任务，都必须进入更严格的审核和真人在环流程。

低风险任务应该容易开始。高风险任务应该难以被滥用。

## 8. 商业与信任

WebAZ 也在探索 agent-native commerce。

买家应该可以把搜索、比较、锁价、下单、查状态和发起争议交给 agent。卖家应该可以上架商品、接单、发货、回答问题，并逐步向买家 agent 暴露自己的服务能力。

但没有信任的商业会变成掠夺。

因此 WebAZ 把订单状态、托管、争议、验证、审计日志、声誉和 claim challenge 都视为协议层问题。目标不是让用户相信一个平台，而是让规则足够可见，使用户、agent、维护者和审核者都能检查到底发生了什么。

信任应该来自状态、证据和责任。

## 9. 治理

WebAZ 从创始人责任开始，但不应该停在那里。

WebAZ 的治理方向是渐进式去中心化。早期阶段需要更强的创始人和维护者责任，因为系统还不成熟，不适合把所有决定都立刻交出去。随着贡献者增加、审计实践成熟、协议不变量稳定，权力应该逐步交给更广泛的贡献者治理。

有些规则是承重墙：

不偷数据。不撒谎。不偏袒。不滥用。不操纵。最小介入。算法透明。参与者可问责。

这些不是口号，而是 review 标准。它们应该影响代码、UI、API、文档、管理员工具、任务板和公开表述。

一个无法保护自身原则的治理系统，只是表演。WebAZ 必须变得更开放，但不能变得更草率。

## 10. 当前状态

WebAZ 当前处于 pre-launch 和持续开发阶段。

代码库已经包含核心协议界面：PWA、MCP、市场交易路径、订单生命周期、争议、贡献者入口、任务协调、管理员面板、审计轨迹和早期治理文档。

但 WebAZ 不应该在真正成熟之前声称已经成熟。指标、用户规模、经济结算状态、去中心化状态和安全保证，都必须在发布时诚实说明。

这个阶段正确的邀请不是：

相信我们，一切都完成了。

而是：

阅读规则。检查代码。从小任务开始。一起把协议建出来。

## 11. 如何参与

参与 WebAZ 可以从很多地方开始。

阅读仓库。阅读贡献者入口。认领一个小任务。提出一个缺失任务。报告 bug。改进文档。补测试。提交 PR。带上你的 agent，但保留一个真实的人作为责任主体。签署 commit。说明你改了什么、为什么改、如何验证。

参与不应该依赖大平台许可。

它需要的是清晰、证据和责任。

WebAZ 尤其应该欢迎那些在别处被忽视的人：被裁员的人，创业者，独立卖家，开源贡献者，大公司之外的建设者，以及那些真实能力被岗位名称遮住的人。

一个协议不能承诺每一份贡献都会变成回报。但它可以拒绝让真诚的工作被抹去。

## 12. 边界

WebAZ 必须谨慎使用自己的语言。

它不应该承诺雇佣关系。不应该承诺奖励。不应该在法律结构尚不存在时承诺所有权。不应该在创始人或维护者控制仍然存在时宣称完全去中心化。不应该暗示 AI 可以决定人的贡献价值。

更强的表达，往往也是更诚实的表达：

WebAZ 正在建设一套基础设施，让贡献变得可见、可署名、可审核，并最终能被建设它的人共同治理。

对于 v0.1，这已经足够。

## 13. 结语

WebAZ 不是让人带着更强的工具，回到旧游戏里继续竞争。

WebAZ 想做的是翻篇。

筑起围墙围猎用户的时代正在过去。参与的时代正在开始。公司和资本不会一夜之间消失，但当人、agent、代码和开放协议一起工作时，它们对机会的垄断会被削弱。

WebAZ 正是为这个转变而建。

每个人参与的方式不一样。有人写代码，有人销售，有人审核，有人翻译，有人测试，有人提建议，也有人只是早早来到这里，帮助这个协议变得更容易被下一个人理解。

这些都算数。

WebAZ 是给不想被单一平台困住的卖家。

给不希望贡献消失在队列里的贡献者。

给被裁员的工程师，创业者，沉默的运营者，翻译者，审核者，测试者，设计者，维护者，以及那些从未得到合适头衔、却一直在解决真实问题的人。

WebAZ 是给带着 agent 的人，而不是用 agent 替代人。

WebAZ 不能承诺每一份贡献都会得到奖励。它能承诺的是：持续建设一个让贡献更难被抹去的世界。

代码可以复现。

而你不能被替代。

从今天开始，你走的每一步，都可以算数。

WebAZ 属于所有建设它的人。
